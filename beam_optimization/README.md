# beam_optimization

Beam optimization tools for the ADIGE line using:

- the real TraceWin simulator;
- neural surrogate models;
- Gymnasium environments;
- model-free and model-based RL algorithms.

The repository is meant to contain the Python code, documentation, notebooks and
small configuration files. Large/local TraceWin assets are intentionally not
stored in normal Git history.

## 1. Clone And Python Setup

From any folder where you want to keep the project:

```bash
git clone https://github.com/boorgoof/rl_beam_optimization.git
cd rl_beam_optimization
```

Create the virtual environment:

```bash
python3 -m venv beam_optimization/.venv
source beam_optimization/.venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -r beam_optimization/requirements.txt
```

When you come back later:

```bash
cd /path/to/rl_beam_optimization
source beam_optimization/.venv/bin/activate
```

Run the project check:

```bash
python -m beam_optimization check
```

This checks imports, dataset/surrogate loading when available, environments and
main RL components. It does not run a real TraceWin simulation.

## 2. Local Files Not Stored On GitHub

Some files are local by design:

```text
beam_optimization/env/tracewin_env/tracewin/TraceWin_workspace/
beam_optimization/env/tracewin_env/tracewin/pyTraceWin_wrapper/TraceWin_program/TraceWin
beam_optimization/env/tracewin_env/tracewin/pyTraceWin_wrapper/run_tracewin_with_permissions.sh
```

These are ignored because they are large, machine-specific, or license-related.

### TraceWin Workspace

Create or copy the physical TraceWin workspace here:

```text
beam_optimization/env/tracewin_env/tracewin/TraceWin_workspace/
```

Expected contents include the project files and field maps, for example:

```text
TraceWin_workspace/
├── condensed_new.ini
├── condensed.dat
├── 16O5.dst
├── sol*.bsr / sol*.bsz
├── sd*.bsx / sd*.bsy / sd*.bsz
├── Mpolo*.esx / Mpolo*.esy / Mpolo*.esz
└── calc/                    # generated by TraceWin, ignored by Git
```

The default project path is configured in:

```text
beam_optimization/config/paths.py
```

Default:

```text
env/tracewin_env/tracewin/TraceWin_workspace/condensed_new.ini
```

### TraceWin Program

Put the licensed TraceWin executable here:

```text
beam_optimization/env/tracewin_env/tracewin/pyTraceWin_wrapper/TraceWin_program/TraceWin
```

Make it executable:

```bash
chmod +x beam_optimization/env/tracewin_env/tracewin/pyTraceWin_wrapper/TraceWin_program/TraceWin
```

If your TraceWin installation uses local license/log files, keep them in the same
folder, for example:

```text
TraceWin_program/
├── TraceWin
├── tracewin_key.log
└── toutatis_key.log
```

Those files are also local and ignored by Git.

## 3. TraceWin Launcher With Permissions

Python does not call `TraceWin_program/TraceWin` directly. It calls this local
launcher:

```text
beam_optimization/env/tracewin_env/tracewin/pyTraceWin_wrapper/run_tracewin_with_permissions.sh
```

This file is intentionally ignored because the correct user/permission setup can
change from machine to machine.

Example launcher using the Linux user `comunian`:

```bash
#!/bin/bash
set -e

SCRIPT_PATH=$(realpath "$0")
SCRIPT_DIR=$(dirname "$SCRIPT_PATH")

ssh -F /dev/null comunian@localhost \
  "cd '$SCRIPT_DIR'; \
   if command -v xvfb-run >/dev/null 2>&1; then \
     xvfb-run -a ./TraceWin_program/TraceWin \"\$@\"; \
   else \
     ./TraceWin_program/TraceWin \"\$@\"; \
   fi" -- "$@"
```

Create it and make it executable:

```bash
nano beam_optimization/env/tracewin_env/tracewin/pyTraceWin_wrapper/run_tracewin_with_permissions.sh
chmod +x beam_optimization/env/tracewin_env/tracewin/pyTraceWin_wrapper/run_tracewin_with_permissions.sh
```

### Required Permission Checks

If you use the `comunian` example, these checks should pass:

```bash
id comunian
ssh -F /dev/null -o BatchMode=yes -o ConnectTimeout=5 comunian@localhost echo OK
```

The SSH command must print `OK` without asking for a password.

The TraceWin user must be able to read the workspace and write to `calc/`:

```bash
mkdir -p beam_optimization/env/tracewin_env/tracewin/TraceWin_workspace/calc
sudo -u comunian test -r beam_optimization/env/tracewin_env/tracewin/TraceWin_workspace/condensed_new.ini
sudo -u comunian test -w beam_optimization/env/tracewin_env/tracewin/TraceWin_workspace/calc
```

### Running TraceWin Not In Trial Mode

To run TraceWin in the licensed/non-trial setup, make sure:

- the real licensed binary is placed at `TraceWin_program/TraceWin`;
- the license/log files required by your installation are available beside it;
- the launcher runs TraceWin as the user for which the license is valid
  (`comunian` in the example above);
- `ssh <tracewin-user>@localhost` works without an interactive password prompt;
- `xvfb-run` is installed if you want headless execution on a server.

You can test the executable manually:

```bash
cd beam_optimization/env/tracewin_env/tracewin/pyTraceWin_wrapper/TraceWin_program
sudo -u comunian DISPLAY=$DISPLAY ./TraceWin
```

If your license is tied to a different Linux user, edit
`run_tracewin_with_permissions.sh` and replace `comunian` with that user.

## 4. Main Commands

CLI entrypoint:

```bash
python -m beam_optimization --help
```

Project check:

```bash
python -m beam_optimization check
```

Quick surrogate training:

```bash
python -m beam_optimization train --quick
```

Skip expensive algorithms:

```bash
python -m beam_optimization train --skip ppo trpo reinforce
```

TraceWin + online fine-tuning:

```bash
python -m beam_optimization train --tracewin --online-finetune
```

Quick benchmark:

```bash
python -m beam_optimization benchmark --quick
```
